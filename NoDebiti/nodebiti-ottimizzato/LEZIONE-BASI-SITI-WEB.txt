════════════════════════════════════════════════════════════
1️⃣ COS'È UN SITO WEB?
════════════════════════════════════════════════════════════

Un sito web è semplicemente un insieme di FILE che il browser 
(Chrome, Firefox, ecc.) sa leggere e mostrare in modo grafico.

╔════════════════════════════════════════════════════════╗
║  PARAGONE CON MICROSOFT WORD:                          ║
║                                                         ║
║  Word legge file .docx → Mostra documento              ║
║  Browser legge file .html → Mostra pagina web          ║
╚════════════════════════════════════════════════════════╝

IMPORTANTE: Un sito web sono SOLO FILE di testo!
            Non c'è magia, sono file come Word o Excel.


════════════════════════════════════════════════════════════
2️⃣ I 3 INGREDIENTI FONDAMENTALI
════════════════════════════════════════════════════════════

Come una casa ha fondamenta, muri e tetto, un sito ha 3 elementi:

┌────────────────────────────────────────────────────────┐
│ 1. HTML  → Lo Scheletro (struttura e contenuto)       │
│ 2. CSS   → La Pelle (colori, stile, design)           │
│ 3. JS    → I Muscoli (interattività e funzionalità)   │
└────────────────────────────────────────────────────────┘


─────────────────────────────────────────────────────────────
🔷 ELEMENTO 1: HTML (HyperText Markup Language)
─────────────────────────────────────────────────────────────

COS'È: Il contenuto e la struttura base del sito

COSA FA:
• Definisce il testo
• Crea titoli, paragrafi, liste
• Inserisce immagini
• Crea link cliccabili
• Definisce form (campi input)

ESEMPIO SEMPLICE:

   <h1>Benvenuto su NoDebiti.it</h1>
   <p>Aiutiamo le persone con i debiti</p>
   <button>Contattaci</button>

ANALOGIA: È come lo scheletro di una persona.
          Definisce cosa c'è (testa, corpo, gambe)
          ma non come appare.


NEL TUO PROGETTO:
• File: index.html
• Contiene: Titoli, form, bottoni, testo del sito


─────────────────────────────────────────────────────────────
🔷 ELEMENTO 2: CSS (Cascading Style Sheets)
─────────────────────────────────────────────────────────────

COS'È: L'aspetto visivo, i colori, lo stile

COSA FA:
• Colori (sfondo verde, testo bianco)
• Dimensioni (bottone grande, testo piccolo)
• Posizionamento (centrare elementi, layout)
• Animazioni (transizioni fluide)
• Font (tipo di carattere)

ESEMPIO SEMPLICE:

   h1 {
     color: green;        ← Titolo verde
     font-size: 32px;     ← Testo grande
   }
   
   button {
     background: blue;    ← Bottone blu
     border-radius: 10px; ← Angoli arrotondati
   }

ANALOGIA: È come vestire una persona.
          Lo scheletro (HTML) dice "c'è una testa",
          il CSS dice "la testa ha capelli biondi e occhi blu".


NEL TUO PROGETTO:
• File: styles.css
• Contiene: Colori verdi, layout moderno, gradienti


─────────────────────────────────────────────────────────────
🔷 ELEMENTO 3: JAVASCRIPT (JS)
─────────────────────────────────────────────────────────────

COS'È: La logica, l'interattività, il "cervello"

COSA FA:
• Reagisce ai click (bottone premuto → apri modal)
• Valida i form (controlla se email è valida)
• Cambia la pagina senza ricaricarla
• Invia email
• Calcola cose

ESEMPIO SEMPLICE:

   button.addEventListener('click', function() {
     alert('Hai cliccato il bottone!');
   });
   
   // Quando clicchi il bottone, appare un messaggio

ANALOGIA: È come il cervello e i muscoli.
          Lo scheletro (HTML) dice "c'è una mano",
          il CSS dice "la mano è abbronzata",
          JS dice "se tocchi qualcosa, la mano si chiude".


NEL TUO PROGETTO:
• File: app.js
• Contiene: Wizard step-by-step, validazione form, invio email


─────────────────────────────────────────────────────────────
COME LAVORANO INSIEME
─────────────────────────────────────────────────────────────

index.html dice:  "C'è un bottone scritto 'Invia'"
styles.css dice:  "Il bottone è verde con angoli arrotondati"
app.js dice:      "Se clicchi il bottone, invia un'email"

╔════════════════════════════════════════════════════════╗
║  BROWSER LEGGE:                                        ║
║                                                         ║
║  1. index.html  → Capisce cosa c'è nella pagina       ║
║  2. styles.css  → Applica i colori e lo stile         ║
║  3. app.js      → Attiva l'interattività              ║
║                                                         ║
║  RISULTATO: Pagina web bella e funzionante!           ║
╚════════════════════════════════════════════════════════╝

3️⃣ DOVE VIVE IL SITO? (Locale vs Online)
════════════════════════════════════════════════════════════

Un sito può vivere in DUE POSTI:


─────────────────────────────────────────────────────────────
🏠 LOCALE (Sul tuo Computer)
─────────────────────────────────────────────────────────────

DOVE: C:\Users\gianl\nodebiti-ottimizzato\
CHI PUÒ VEDERLO: Solo tu
COME SI APRE: Doppio click su index.html
URL: file:///C:/Users/gianl/nodebiti/index.html

✅ PRO:
   • Gratis
   • Modifichi e vedi subito i cambiamenti
   • Non serve Internet per lavorarci

❌ CONTRO:
   • Solo tu puoi vederlo
   • Non è pubblico su Internet
   • Se spegni il PC, il sito non c'è


─────────────────────────────────────────────────────────────
🌍 ONLINE (Su Internet)
─────────────────────────────────────────────────────────────

DOVE: Su un server (computer sempre acceso connesso a Internet)
CHI PUÒ VEDERLO: Chiunque nel mondo
COME SI APRE: https://nodebiti.netlify.app
URL: https://nomesito.com

✅ PRO:
   • Chiunque può visitarlo
   • Sempre online 24/7
   • Accessibile da telefono, tablet, ovunque

❌ CONTRO:
   • Serve un "hosting" (vedi dopo)
   • Può costare (ma esistono opzioni gratuite!)

4️⃣ HOSTING: La Casa del Sito
════════════════════════════════════════════════════════════

ANALOGIA: Se il sito sono i tuoi mobili, l'hosting è la casa
          dove metti i mobili.

COS'È: Un computer sempre acceso, connesso a Internet,
       che "ospita" (host) i tuoi file HTML/CSS/JS.

COSA FA:
1. Conserva i tuoi file
2. Li rende accessibili 24/7
3. Li invia a chi visita il sito


─────────────────────────────────────────────────────────────
COME FUNZIONA L'HOSTING
─────────────────────────────────────────────────────────────

   TU                   HOSTING              VISITATORE
    │                      │                      │
    │  Carico i file       │                      │
    ├──────────────────────>                      │
    │                      │                      │
    │                      │  Chiede la pagina   │
    │                      <──────────────────────┤
    │                      │                      │
    │                      │  Invia index.html   │
    │                      ├──────────────────────>
    │                      │                      │
    │                      │  ✅ Vede il sito!    │


─────────────────────────────────────────────────────────────
TIPI DI HOSTING
─────────────────────────────────────────────────────────────

1. HOSTING CONDIVISO (Shared Hosting)
   • Come vivere in un appartamento condiviso
   • Molti siti sullo stesso server
   • €3-10/mese
   • Es: Aruba, SiteGround

2. VPS (Virtual Private Server)
   • Come avere un appartamento tutto tuo
   • Più potenza, più controllo
   • €15-50/mese
   • Es: DigitalOcean, Linode

3. HOSTING STATICO (Per siti come il tuo)
   • Per siti HTML/CSS/JS puri (senza database)
   • SPESSO GRATUITO!
   • Es: Netlify, Vercel, GitHub Pages
   
   ⭐ QUESTO È QUELLO CHE USI TU!


─────────────────────────────────────────────────────────────
HOSTING GRATUITI (Perfetti per te!)
─────────────────────────────────────────────────────────────

NETLIFY:
• 100 GB bandwidth/mese gratis
• Deploy con drag & drop
• HTTPS automatico
• ⭐ Il più semplice

VERCEL:
• 100 GB bandwidth

GITHUB PAGES:

CLOUDFLARE PAGES:

5️⃣ DOMINIO: L'Indirizzo del Sito
════════════════════════════════════════════════════════════

ANALOGIA: Se l'hosting è la casa, il dominio è l'indirizzo
          della casa (es: Via Roma 123).

COS'È: Un nome facile da ricordare che punta al tuo sito.


─────────────────────────────────────────────────────────────
STRUTTURA DI UN DOMINIO
─────────────────────────────────────────────────────────────

   https://www.nodebiti.it/pagina
   
   ├─ https://  → Protocollo (sicuro)
   ├─ www.      → Sottodominio (opzionale)
   ├─ nodebiti  → Nome che scegli tu
   ├─ .it       → Estensione (TLD)
   └─ /pagina   → Percorso interno


─────────────────────────────────────────────────────────────
PARTI DEL DOMINIO
─────────────────────────────────────────────────────────────

1. NOME (Second-Level Domain)
   • nodebiti ← Questo lo scegli tu
   • Può contenere: lettere, numeri, trattini
   • Non può: spazi, caratteri speciali

2. ESTENSIONE (Top-Level Domain - TLD)
   • .it  → Italia
   • .com → Commerciale (il più comune)
   • .org → Organizzazioni
   • .io  → Tech/Startup
   • .net → Network
   
   Prezzo: €8-20/anno


─────────────────────────────────────────────────────────────
COME FUNZIONA UN DOMINIO
─────────────────────────────────────────────────────────────

Un dominio è semplicemente un "soprannome" per un indirizzo IP.

   SENZA DOMINIO:  http://192.168.1.45  (difficile da ricordare)
   CON DOMINIO:    https://nodebiti.it  (facile!)

Dietro le quinte:
   nodebiti.it → punti a → 75.2.60.5 (IP del server Netlify)

Come una rubrica telefonica:
   "Marco" → 333-1234567
   "nodebiti.it" → 75.2.60.5

─────────────────────────────────────────────────────────────

7️⃣ ELEMENTI EXTRA (Opzionali ma utili)
════════════════════════════════════════════════════════════

Oltre ai 3 fondamentali (HTML/CSS/JS), ci sono altri elementi:


─────────────────────────────────────────────────────────────
🔷 BACKEND (Server-side)
─────────────────────────────────────────────────────────────

COS'È: Codice che gira SUL SERVER, non nel browser

LINGUAGGI:
• PHP (molto usato)
• Node.js (JavaScript lato server)
• Python (Django, Flask)
• Ruby (Rails)

QUANDO SERVE:
• Database (salvare dati utenti)
• Login/registrazione
• Pagamenti
• Logica complessa server-side

NEL TUO CASO:
❌ NON SERVE! Il tuo sito è "frontend-only"
   (tutto funziona nel browser del visitatore)


─────────────────────────────────────────────────────────────
🔷 DATABASE
─────────────────────────────────────────────────────────────

COS'È: Dove salvi i dati permanentemente

TIPI:
• MySQL / PostgreSQL (relazionali)
• MongoDB (NoSQL)
• Firebase (cloud)

QUANDO SERVE:
• E-commerce (prodotti, ordini)
• Blog (articoli, commenti)
• Social network (utenti, post)

NEL TUO CASO:
❌ NON SERVE! I dati del form li invii via email
   (con EmailJS)


─────────────────────────────────────────────────────────────
🔷 CMS (Content Management System)
─────────────────────────────────────────────────────────────

COS'È: Interfaccia per gestire contenuti senza toccare codice

ESEMPI:
• WordPress (il più famoso)
• Joomla
• Drupal

QUANDO SERVE:
• Blog frequenti
• Siti con molti contenuti
• Più persone devono aggiornare il sito

NEL TUO CASO:
❌ NON SERVE! Hai un sito statico semplice
   Se devi modificare, cambi direttamente i file


─────────────────────────────────────────────────────────────
🔷 CDN (Content Delivery Network)
─────────────────────────────────────────────────────────────

COS'È: Rete di server in tutto il mondo che velocizza il sito

COME FUNZIONA:
   Visitatore in Italia → Server CDN in Milano (veloce!)
   Visitatore in USA    → Server CDN in New York (veloce!)

NEL TUO CASO:
✅ Netlify lo include GRATIS!
   Il tuo sito è già su CDN globale


─────────────────────────────────────────────────────────────
🔷 SSL/HTTPS (Certificato Sicurezza)
─────────────────────────────────────────────────────────────

COS'È: Il "lucchetto" nel browser, connessione criptata

DIFFERENZA:
   http://  → Non sicuro ❌
   https:// → Sicuro ✅

NEL TUO CASO:
✅ Netlify lo attiva AUTOMATICAMENTE!
   Certificato Let's Encrypt gratuito


─────────────────────────────────────────────────────────────
🔷 EMAIL SERVICE
─────────────────────────────────────────────────────────────

COS'È: Servizio per inviare email dal sito

COME FUNZIONA:
   Utente compila form → EmailJS → Invia email → Tua inbox

NEL TUO CASO:
✅ Usi EmailJS (già integrato!)
   200 email/mese gratis
